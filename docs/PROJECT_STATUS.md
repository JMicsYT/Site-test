## 1. Полностью сделано

### Безопасность и надёжность
| Пункт | Описание |
|-------|----------|
| env.example | Файл `backend/env.example` со всеми переменными и комментариями. |
| Редирект после входа (`?next=`) | Поддержка параметра `next` при логине с проверкой внутреннего URL. |
| Сброс пароля (SMTP) | Настройка по `EMAIL_*`, документация; без SMTP письма в консоль. |
| Ограничение по email_verified | Заказы/корзина только при подтверждённом email; кнопка «Отправить письмо повторно». |
| Callback платежей | Проверка секрета (X-Payment-Secret / body), идемпотентность, только PENDING. |
| Идемпотентность оплаты и выдачи | Повторный success не дублирует выдачу. |
| Валидация корзины и заказов | Лимит qty (1–99), проверка товара и статуса, отсев некорректных данных. |
| Rate limit входа | По IP и пользователю, при превышении — HTML-страница с сообщением, не 429 JSON. |
| DEBUG не в шаблонах | Снижение риска утечки в проде. |
| HSTS при DEBUG | Отключён при DEBUG=True для локальной разработки. |
| Логирование действий в админке | Смена статуса заказа, удаление товара/категории пишутся в SecurityEvent. |

### Функциональность и UX
| Пункт | Описание |
|-------|----------|
| Удаление из корзины | Кнопка «Удалить», изменение количества на странице корзины. |
| Пагинация каталога | 24 товара на страницу, ссылки с сохранением фильтров в URL. |
| Пагинация API каталога | DRF ProductViewSet с PageNumberPagination (24 на страницу). |
| Кнопка «Скопировать» в ЛК | Для выданных цифровых товаров — копирование в буфер. |
| Подтверждение при выходе | Диалог «Вы уверены, что хотите выйти?». |
| Футер внизу экрана | Flex-верстка: футер прижат к низу на коротких страницах. |
| Модерация отзывов | Статусы отзывов (на модерации / опубликован / скрыт), фильтр и кнопки в dashboard; на сайте только опубликованные. |
| Экспорт заказов | Кнопка «Экспорт в CSV» на странице заказов с теми же фильтрами (статус, даты). |

### Визуал и контент
| Пункт | Описание |
|-------|----------|
| Единый стиль ошибок форм | Классы `.errorlist` и `.field-error` в main.css (красный текст под полем). |
| Страница 500 | Нейтральный текст («Временная ошибка», «Попробуйте позже»), без технических деталей. |
| Мета-теги SEO | Блок `{% block meta %}` в base.html; description и keywords на главной и в каталоге. |
| Хлебные крошки | Были реализованы на сайте; убраны по запросу (остались только в админке). |

### Инфраструктура и документация
| Пункт | Описание |
|-------|----------|
| Кеш | Кеш категорий в каталоге, кеш популярных товаров на главной. |
| Статика в проде | В DEPLOY_CHECKLIST и RUN_LOCAL указаны collectstatic и раздача /static/ через Nginx. |
| LOGGING | В settings.py настроен LOGGING: консоль, INFO для apps.*, WARNING для остальных. |
| Health-check | Эндпоинт `/health/` (проверка БД), 200/503. |
| README | Перечень документов, переменные окружения (PAYMENT_CALLBACK_SECRET, PROJECT_NAME и др.). |
| Пример .env | backend/env.example; в RUN_LOCAL описано копирование в .env. |
| Чек-лист перед релизом | docs/DEPLOY_CHECKLIST.md (DEBUG, SECRET_KEY, ALLOWED_HOSTS, HTTPS, collectstatic, бэкапы и т.д.). |

### Стек и базовая функциональность
| Пункт | Описание |
|-------|----------|
| Backend | Django 5, DRF, PostgreSQL. |
| Фронт | Шаблоны + CSS, тёмная тема ShoShop. |
| Docker, Nginx, Gunicorn | Конфигурация есть (docker-compose, config/nginx). |
| Каталог, корзина, заказы, оплата | Реализованы; оплата через абстракцию провайдера + заглушка. |
| Админка (dashboard) | Товары, категории, заказы, пользователи, отзывы, настройки, журнал безопасности. |
| API | Каталог (чтение), заказы (по пользователю), аккаунты, платежи (callback). |
| Тесты | Есть тесты: orders, catalog (API), core, payments, **accounts**, **dashboard**, расширенные **payments callback** (403, секрет, идемпотентность). |
| Валидация форм в админке | Slug (формат, уникальность), цена ≥ 0, обязательные поля (name, category) в формах категории и товара. |
| Страница «Мои заказы» | Отдельный URL `accounts/orders/`, шаблон со списком заказов, ссылка в сайдбаре ЛК. |

## 3. Частично / наполовину

| Пункт | Что есть | Чего нет |
|-------|----------|----------|
| Favicon | В base.html используется inline SVG‑иконка (буква S на фиолетовом фоне) в виде data URI — без 404. | При желании можно заменить на свой `favicon.ico` в `static/`. |
| Celery | В проекте: shoshop/celery.py, accounts/tasks.py (отправка писем верификации). В requirements.txt — celery и redis. | При запуске без установленного celery (например из корня без venv) возможна ошибка импорта; для полной работы нужны воркер и брокер (Redis). |
| Хлебные крошки | В админке (dashboard) крошки есть и работают; на карточке товара — «Главная / Каталог / Товар». | На остальных страницах публичного сайта не выводятся. |

---

**Совместимость с Python 3.14:** при запуске тестов под Python 3.14 возможна ошибка `AttributeError: 'super' object has no attribute 'dicts'` при копировании контекста шаблонов в тестовом клиенте Django. В тестах dashboard для части проверок используется RequestFactory вместо client. Для полного прохождения тестов рекомендуется Python 3.11 или 3.12 (или обновление Django после выхода исправления).
