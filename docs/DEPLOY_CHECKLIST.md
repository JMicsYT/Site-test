# Чек-лист перед релизом ShoShop

Проверьте перед выкладкой на прод или сдачей.

## Безопасность и настройки Django

- [ ] **DEBUG=false** — в `.env` установлен `DJANGO_DEBUG=false`.
- [ ] **SECRET_KEY** — уникальный ключ, не из репозитория (не оставляйте значение из `env.example`).
- [ ] **ALLOWED_HOSTS** — в `DJANGO_ALLOWED_HOSTS` указаны только нужные домены (через запятую).
- [ ] **HTTPS** — сайт доступен по HTTPS; в проде задан **DJANGO_SECURE_SSL_REDIRECT=true** для редиректа HTTP → HTTPS.
- [ ] **PAYMENT_CALLBACK_SECRET** — задан надёжный секрет для проверки callback платёжного провайдера.

## Статика и медиа

- [ ] **collectstatic** — перед запуском (или в CI/CD) выполнен `python manage.py collectstatic --noinput`. Статика отдаётся из одной директории (например `STATIC_ROOT`).
- [ ] **Nginx** — в документации по деплою указано, что Nginx отдаёт `/static/` из каталога со статикой (например `/var/www/static/` или путь из `STATIC_ROOT`). Конфиг примера: `config/nginx/shoshop.conf` (location `/static/` и `/media/`).

## База данных

- [ ] **Миграции** — применены: `python manage.py migrate`.
- [ ] **Бэкапы** — настроено регулярное резервное копирование БД (cron, облачный бэкап и т.п.). Пример в `docs/README.md` (раздел «Резервное копирование БД»).

## Инфраструктура и мониторинг

- [ ] **Health-check** — эндпоинт `/health/` возвращает 200 при доступной БД и 503 при ошибке (для оркестрации/проверок).
- [ ] **Логирование** — в `settings.py` настроен `LOGGING` (логи приложения в консоль/файл по необходимости).

## Опционально

- [ ] **Favicon** — по умолчанию используется inline SVG в base.html; при необходимости можно заменить на свой `backend/static/favicon.ico`.
- [ ] **Celery/Redis** — если используются фоновые задачи, брокер и воркеры запущены и доступны.

После прохода по пунктам можно считать конфигурацию готовой к продакшену.
