# ShoShop

**Защищённая веб-платформа продажи цифровых товаров** — дипломный/учебный проект на Django: каталог, корзина, оплата (заглушка и расширяемый провайдер), выдача цифровых ключей, личный кабинет, кастомная админ-панель, поддержка, аналитика и набор мер безопасности уровня production.

Этот документ — **основная техническая документация**: что внутри проекта, зачем сделано именно так, как поднять локально, как развернуть на хосте, какие переменные окружения критичны и как устроены подсистемы.

Дополнительные материалы в каталоге `[docs/](docs/)`: чек-лист деплоя, детальный разбор безопасности OWASP, запуск без Docker.

## 1. Назначение и возможности

### Зачем проект

Платформа демонстрирует полный цикл **e-commerce для цифровых товаров** (лицензии, ключи, контент) с упором на:

- **Безопасность**: шифрование чувствительных данных, 2FA, CSP, rate limiting, аудит, безопасные ссылки на скачивание, проверка платёжных callback и т.д.
- **Функциональность**: не только «витрина», но и промокоды, кошелёк, рефералы, отзывы, избранное, сравнение, умный поиск, фильтры каталога, уведомления.
- **Администрирование**: собственный дашборд (не только Django Admin) с графиками, экспортом заказов (CSV/XLSX/PDF), массовыми операциями, таймлайном заказа, модерацией отзывов, поддержкой.

### Что умеет пользователь


| Область         | Возможности                                                                                                                    |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| Каталог         | Список, детальная страница, категории, расширенные фильтры (цена, скидка, новинки, рейтинг, сортировки), AJAX-обновление сетки |
| Поиск           | Автодополнение по API (`/api/catalog/autocomplete/`)                                                                           |
| Покупка         | Корзина, быстрый «Купить», оформление, промокод, списание с внутреннего кошелька                                               |
| Цифровые товары | Выдача через **одноразовые подписанные ссылки** с TTL и лимитом скачиваний; значения в БД могут быть **зашифрованы** (Fernet)  |
| Личный кабинет  | Профиль, заказы, отмена/повтор оплаты (для подходящих статусов), 2FA, избранное, кошелёк, реферальная программа                |
| Социальное      | Отзывы (после покупки, модерация), избранное, сравнение до 4 товаров                                                           |
| Поддержка       | Тикеты и сообщения; уведомления при ответе админа                                                                              |
| Уведомления     | Внутри сайта + WebSocket (`/ws/notifications/`) при настроенном ASGI и Redis                                                   |


### Что умеет администратор

- Кастомный `**/dashboard/`** (доступ `is_staff`): товары, категории, пользователи, заказы с фильтрами и **массовой сменой статуса**, экспорт заказов, отзывы, поддержка, настройки сайта, журнал событий безопасности.
- Графики на главной админки (Chart.js), таймлайн заказа (`OrderEvent`), внутренние комментарии к заказу.
- Стандартный `**/admin/`** Django для низкоуровневого доступа к моделям.
- Опционально: **Telegram-алерты** о новых оплатах и сообщениях клиентов в поддержку.

## 2. Архитектура и стек


| Компонент     | Технология                                                                                                                             | Зачем                                              |
| ------------- | -------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------- |
| Язык          | Python 3.11+ (в Docker образе — 3.11; локально допустимы совместимые версии)                                                           | Среда выполнения                                   |
| Фреймворк     | **Django 5**                                                                                                                           | MVT, ORM, админка, формы, auth                     |
| API           | **Django REST Framework**                                                                                                              | JSON API для части сущностей                       |
| БД            | **PostgreSQL только**                                                                                                                  | Надёжность, JSON, без SQLite по требованию проекта |
| Кеш           | LocMem по умолчанию; опционально **Redis** (`CACHE_URL`)                                                                               | Rate limit, масштабируемый кеш                     |
| Очереди       | **Celery** + Redis                                                                                                                     | Фоновые задачи (при необходимости расширения)      |
| Real-time     | **Django Channels** + Redis channel layer (`CHANNELS_REDIS_URL`)                                                                       | WebSocket уведомлений                              |
| HTTP (prod)   | **Gunicorn** (WSGI) в `docker-compose`                                                                                                 | Традиционный деплой                                |
| ASGI          | **Daphne** в зависимостях                                                                                                              | Запуск с WebSocket (см. раздел про WS)             |
| Frontend      | Шаблоны Django + **CSS** (`static/css/main.css`, `dashboard.css`) + **JS** (поиск, фильтры, избранное, сравнение, уведомления, toasts) | Без отдельного SPA                                 |
| Reverse proxy | **Nginx** (пример в `config/nginx/`)                                                                                                   | HTTPS, статика, медиа, заголовки                   |
| Контейнеры    | **Docker Compose**                                                                                                                     | db, redis, web, celery, nginx                      |


**Поток запроса (упрощённо):** браузер → Nginx → Gunicorn (Django WSGI) → PostgreSQL / кеш / файлы. WebSocket при продакшене требует отдельного ASGI-процесса и проксирования `Upgrade` (см. [§14](#14-websocket-и-уведомления-в-реальном-времени)).

## 4. Требования к окружению

- **PostgreSQL** 14+ (в Compose — 16).
- **Python** 3.11+ рекомендуется (см. `Dockerfile`).
- Для полного стека: **Redis** (кеш, Celery, Channels).
- Для продакшена: **Nginx** или аналог, TLS-сертификат.

**SQLite в проекте намеренно не используется:** в `settings.py` стоит проверка `ENGINE == postgresql`.

## 5. Локальная разработка

### 5.1. Клонирование и виртуальное окружение

Из корня репозитория:

```powershell
cd F:\path\to\Site-test
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r backend\requirements.txt
```

Убедитесь, что установлены все зависимости, включая `**cryptography**` (нужен для `apps.core.crypto` и шифрования цифровых значений). Если видите `ModuleNotFoundError: No module named 'cryptography'`, выполните:

```powershell
pip install -r backend\requirements.txt
```

### 5.2. PostgreSQL

Создайте БД и пользователя (пример):

```sql
CREATE USER shoshop WITH PASSWORD 'your_password';
CREATE DATABASE shoshop OWNER shoshop;
```

### 5.3. Файл окружения

Скопируйте шаблон:

```powershell
copy backend\env.example backend\.env
```

Отредактируйте `backend\.env`: как минимум `POSTGRES_*`, `DJANGO_SECRET_KEY`, для разработки часто задают `DJANGO_DEBUG=true`, `DJANGO_SECURE_SSL_REDIRECT=false`, `SECURE_PROXY_SSL_HEADER=false` (чтобы тесты и `runserver` не ловили лишние редиректы).

### 5.4. Миграции и суперпользователь

```powershell
python manage.py migrate
python manage.py createsuperuser
```

Опционально демо-данные:

```powershell
python manage.py seed_demo
```

### 5.5. Запуск сервера разработки

```powershell
python manage.py runserver 127.0.0.1:8000
```

Откройте `http://127.0.0.1:8000/`. Админ-дашборд: `http://127.0.0.1:8000/dashboard/` (нужен staff).

**Важно:** `runserver` использует **WSGI**; WebSocket-уведомления в реальном времени для полной проверки лучше запускать через **Daphne** (см. §14).

### 5.6. Тесты

```powershell
python manage.py test apps.accounts apps.catalog apps.core apps.orders apps.payments apps.dashboard
```

## 6. Развёртывание через Docker

В `docker-compose.yml` описаны сервисы: `web` (Gunicorn), `db` (Postgres), `redis`, `celery`, `celery-beat`, `nginx`.

### 6.1. Переменные для Compose

`docker-compose` подключает `**env_file: .env**` от **корня репозитория** (рядом с `docker-compose.yml`). Практично скопировать значения из `backend/env.example` в **корневой** `.env` или смонтировать/продублировать нужные ключи, чтобы контейнер `web` получил `POSTGRES_HOST=db`, `CELERY_BROKER_URL=redis://redis:6379/0` и т.д.

Внутри образа рабочий каталог — `/app` (содержимое `backend/`). Файл `backend/.env` на хосте попадает в том при бинде `./backend:/app`, но переменные из **корневого** `.env` всё равно инжектятся Docker Compose в окружение процесса.

### 6.2. Сборка и запуск

```bash
docker compose build
docker compose up -d
```

### 6.3. Первичная настройка в контейнере

```bash
docker compose exec web python manage.py migrate
docker compose exec web python manage.py collectstatic --noinput
docker compose exec web python manage.py createsuperuser
```

Nginx в примере слушает 80/443; статика и медиа монтируются из томов. Подробности — в `config/nginx/shoshop.conf`.

### 6.4. Ограничение текущего Compose для WebSocket

Команда `web` — **Gunicorn WSGI**. Для **полноценных WebSocket** в проде добавьте сервис с **Daphne** (или другим ASGI-сервером) и настройте Nginx на `proxy_http_version 1.1`, `Upgrade`, `Connection "upgrade"` для пути `/ws/`. См. §14.

## 7. Развёртывание на VPS / хостинге

Общая схема «классического» VPS:

1. Установить **PostgreSQL**, **Redis** (по желанию), **Nginx**.
2. Создать системного пользователя для приложения, каталог `/srv/shoshop` (пример).
3. Развернуть код, виртуальное окружение, `pip install -r backend/requirements.txt`.
4. Настроить `**backend/.env`** (см. §10): `DEBUG=false`, сильный `SECRET_KEY`, корректные `ALLOWED_HOSTS`, `DJANGO_SECURE_SSL_REDIRECT=true` при отдаче сайта только по HTTPS, при прокси — `SECURE_PROXY_SSL_HEADER=true`.
5. `python manage.py migrate`, `collectstatic`.
6. Запуск приложения:
  - **Gunicorn** (WSGI), например:  
   `gunicorn shoshop.wsgi:application --bind 127.0.0.1:8000 --workers 4`
  - Отдельно при необходимости **Daphne** для ASGI/WebSocket на другом порту.
7. **Nginx**: `proxy_pass` на Gunicorn; отдельные `location` для `/static/` и `/media/`; SSL через Let’s Encrypt.
8. **Systemd** unit-файлы для `gunicorn`, `daphne`, `celery`, `celery-beat`.
9. Регулярные **бэкапы БД** (cron + `pg_dump` или managed PostgreSQL).

Пошаговый гайд по серверу: [`docs/DEPLOY_GUIDE.md`](docs/DEPLOY_GUIDE.md).  
Короткий чек-лист перед релизом: [`docs/DEPLOY_CHECKLIST.md`](docs/DEPLOY_CHECKLIST.md).

## 8. База данных PostgreSQL

- Параметры подключения задаются через `POSTGRES_*` в `.env`.
- Если `POSTGRES_HOST=db` (Docker), но хост `db` не резолвится (локальный запуск без Compose), в `settings.py` выполняется попытка переключиться на `localhost`.
- **Миграции:** только через `python manage.py migrate`; не править историю миграций в общей ветке без необходимости.

### Резервное копирование

В проекте есть команда `**dump_db`**:

```bash
python manage.py dump_db
python manage.py dump_db --out ./backups
```

Дополнительно настройте внешние бэкапы и хранение вне сервера.

## 10. Переменные окружения

Ниже — объединённый справочник. Полный шаблон см. `**backend/env.example`**. Имена чувствительны к регистру в `.env`.

### 10.1. Django


| Переменная             | Назначение                                                        |
| ---------------------- | ----------------------------------------------------------------- |
| `DJANGO_SECRET_KEY`    | Секрет подписи сессий, CSRF и т.д. Обязателен уникальный в проде. |
| `DJANGO_DEBUG`         | `true`/`false`. В проде — только `false`.                         |
| `DJANGO_ALLOWED_HOSTS` | Список хостов через запятую.                                      |
| `PROJECT_NAME`         | Отображаемое имя проекта.                                         |


### 10.2. База данных


| Переменная                                          | Назначение                                   |
| --------------------------------------------------- | -------------------------------------------- |
| `POSTGRES_DB`, `POSTGRES_USER`, `POSTGRES_PASSWORD` | Учётные данные БД.                           |
| `POSTGRES_HOST`                                     | `localhost` локально, `db` в Docker Compose. |
| `POSTGRES_PORT`                                     | Обычно `5432`.                               |
| `DB_CONN_MAX_AGE`, `DB_CONNECT_TIMEOUT`             | Опционально (см. `settings.py`).             |


### 10.3. Кеш и брокеры


| Переменная                                   | Назначение                                                                  |
| -------------------------------------------- | --------------------------------------------------------------------------- |
| `CACHE_URL`                                  | Если задан — Redis-кеш (например `redis://127.0.0.1:6379/1`). Иначе LocMem. |
| `CELERY_BROKER_URL`, `CELERY_RESULT_BACKEND` | Redis для Celery.                                                           |


### 10.4. Каналы (WebSocket)


| Переменная           | Назначение                                                                                                                   |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| `CHANNELS_REDIS_URL` | Если задан — Redis как channel layer для Django Channels. Иначе InMemory (удобно для тестов, плохо для нескольких воркеров). |


### 10.5. Почта и домен


| Переменная                                                                            | Назначение                                    |
| ------------------------------------------------------------------------------------- | --------------------------------------------- |
| `EMAIL_HOST`, `EMAIL_PORT`, `EMAIL_USE_TLS`, `EMAIL_HOST_USER`, `EMAIL_HOST_PASSWORD` | SMTP. Без `EMAIL_HOST` письма идут в консоль. |
| `DEFAULT_FROM_EMAIL`                                                                  | Адрес отправителя.                            |
| `DEFAULT_DOMAIN`                                                                      | Домен для ссылок в письмах.                   |
| `REQUIRE_EMAIL_VERIFIED_FOR_PURCHASE`                                                 | Требовать подтверждённый email для покупок.   |


### 10.6. Безопасность и криптография


| Переменная                                                 | Назначение                                                                                                    |
| ---------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| `FIELD_ENCRYPTION_KEY`                                     | Ключ Fernet для шифрования `DigitalItem.value`. **Потеря/смена ключа = потеря расшифровки.**                  |
| `DOWNLOAD_LINK_SECRET`                                     | Секрет одноразовых ссылок (по умолчанию может наследоваться от `SECRET_KEY` — лучше задать отдельно в проде). |
| `DOWNLOAD_LINK_TTL`, `DOWNLOAD_LINK_MAX_USES`              | TTL (сек) и лимит открытий ссылки.                                                                            |
| `HIBP_CHECK_ENABLED`                                       | Проверка паролей через Have I Been Pwned (k-anonymity).                                                       |
| `ACCOUNT_LOCKOUT_MAX_FAILURES`, `ACCOUNT_LOCKOUT_DURATION` | Блокировка учётки после неудачных попыток.                                                                    |
| `LOGIN_RATE_LIMIT`, `LOGIN_RATE_WINDOW`                    | Rate limit входа/регистрации по IP (middleware).                                                              |
| `CSP_ENABLED`                                              | Включение заголовка Content-Security-Policy.                                                                  |
| `DJANGO_SECURE_SSL_REDIRECT`                               | Редирект HTTP→HTTPS (осторожно при тестах по HTTP).                                                           |
| `SECURE_PROXY_SSL_HEADER`                                  | `true`, если за Nginx TLS и Django видит HTTP, но нужно считать запросы «безопасными».                        |
| `PAYMENT_CALLBACK_REPLAY_WINDOW`                           | Окно валидности timestamp для anti-replay на callback оплаты.                                                 |


### 10.7. Платежи


| Переменная                              | Назначение                                                                                         |
| --------------------------------------- | -------------------------------------------------------------------------------------------------- |
| `PAYMENT_PROVIDER_CLASS`                | Python-путь к классу провайдера (например заглушка `apps.payments.providers.StubPaymentProvider`). |
| `PAYMENT_API_KEY`, `PAYMENT_API_SECRET` | Ключи провайдера.                                                                                  |
| `PAYMENT_CALLBACK_URL`                  | Публичный URL endpoint callback.                                                                   |
| `PAYMENT_CALLBACK_SECRET`               | Общий секрет для проверки подписи callback (обязателен в проде).                                   |


### 10.8. Telegram (опционально)


| Переменная                       | Назначение                                     |
| -------------------------------- | ---------------------------------------------- |
| `TELEGRAM_NOTIFICATIONS_ENABLED` | `true` для включения.                          |
| `TELEGRAM_BOT_TOKEN`             | Токен бота.                                    |
| `TELEGRAM_ADMIN_CHAT_ID`         | Куда слать сообщения (chat id администратора). |


### 10.9. Реферальная программа


| Переменная              | Назначение                                                           |
| ----------------------- | -------------------------------------------------------------------- |
| `REFERRAL_BONUS_AMOUNT` | Сумма бонуса пригласившему при первой оплате реферала (руб., целое). |


Захват реферальной ссылки: GET-параметр `?ref=CODE` сохраняется в сессии (`ReferralCaptureMiddleware`).

## 14. WebSocket и уведомления в реальном времени

- Маршрут WebSocket: `**/ws/notifications/`** (см. `apps/notifications/routing.py`).
- Клиент подключается из `static/js/notifications.js` к `ws(s)://<хост>/ws/notifications/`.
- **Channel layer:** при `CHANNELS_REDIS_URL` используется Redis; иначе InMemory (один процесс, не для кластера).
- **ASGI:** `shoshop/asgi.py` объединяет HTTP (Django) и WebSocket через `ProtocolTypeRouter`.

**Запуск для разработки с WebSocket:**

```bash
cd backend
daphne -b 127.0.0.1 -p 8000 shoshop.asgi:application
```

В проде Nginx должен проксировать WebSocket на процесс Daphne (или аналог).

## 15. Платежи

- Провайдер выбирается классом из `PAYMENT_PROVIDER` в настройках.

- Для демонстрации используется заглушка; интеграция реального эквайринга — замена/наследование провайдера и настройка секретов.

## 16. Безопасность

Краткий перечень (детали — в `[docs/SECURITY.md](docs/SECURITY.md)`):

- CSRF, HttpOnly/SameSite cookies, HSTS и опции SSL в проде  
- CSP, security headers, ограничение iframe  
- Rate limiting входа, блокировка аккаунта, защита от перечисления пользователей где применимо  
- 2FA (TOTP), резервные коды  
- Шифрование чувствительных полей, одноразовые ссылки на выдачу  
- Аудит (`SecurityEvent`, логирование)  
- Валидация загрузок (аватары и др.)  
- Soft-delete, OwnerRequiredMixin, журнал админ-действий

**Секреты никогда не коммитьте.** Используйте `.env` и секрет-хранилища на хостинге.

## 20. Типичные проблемы


| Симптом                                      | Что проверить                                                                                                                                     |
| -------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ModuleNotFoundError: cryptography`          | Активирован ли нужный venv, `pip install -r backend/requirements.txt`                                                                             |
| `connection to server ... failed` (Postgres) | `POSTGRES_HOST`, порт, запущен ли Postgres, пароль                                                                                                |
| Все ответы **301** в тестах                  | `DJANGO_SECURE_SSL_REDIRECT` / прокси-заголовки в `.env` для dev                                                                                  |
| WebSocket не подключается                    | Daphne/ASGI, Redis channel layer, Nginx Upgrade, `ALLOWED_HOSTS` / Origin                                                                         |
| Графики в дашборде не грузятся               | CSP и доступность `cdn.jsdelivr.net`, консоль браузера                                                                                            |
| `.env` не подхватывается                     | Файл должен быть в `backend/.env` или доступен через `python-dotenv` пути из `settings.py`; для Docker — переменные из корневого `.env` у Compose |
